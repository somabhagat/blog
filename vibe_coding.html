In payments we deal in ISO 8583, PCI-DSS, and zero-error-tolerance clearing cycles. "Vibes" have no place here — until now. Here is what happens when you apply the principles of agentic AI development to the most precision-driven industry in tech.

I recently finished reading the book Vibe Coding: Building Production-Grade Software With GenAI, Chat, Agents, and Beyond by Gene Kim and Steve Yegge. The title is provocative by design — and in fintech, it lands like a provocation. We do not do "vibes." We do compliance frameworks, audit trails, and settlement reconciliation. In payments specifically, the architecture and discovery layer is where the most time has historically been lost. Legacy banking cores are not well-documented. Mapping a transaction flow across a ten-year-old codebase used to take months. With Claude Code and Sourcegraph working in tandem, you can achieve precise, targeted changes to live systems without the risk of touching something you did not mean to.

The UI/UX validation layer is equally powerful for payments product work. Before committing to a full sprint cycle on a merchant payout dashboard or a dispute resolution workflow, you can build a functional prototype in Lovable or Bolt.new in hours and test whether the interaction model actually works. You are testing the vibe before you write a single line of production code.

The missing link: Model Context Protocol

MCP is the infrastructure layer that transforms AI from a chatbot into a genuine fintech agent. It is the piece the book points toward but the industry is only now building.

Every payments organisation I have worked with — from large schemes to specialist PSPs — has the same structural problem: data siloed across systems that were never designed to talk to each other. SQL databases holding transaction records. Jira tracking exceptions. Confluence storing runbooks. API logs scattered across environments. The AI tools we use today can reason brilliantly — but only over what they can see.

Model Context Protocol changes this. MCP acts as a universal, secure bridge that allows AI agents to read context from these disparate systems in real time. It is not a data warehouse or an ETL pipeline. It is a live connection layer — governed, auditable, and permissioned — that gives an AI agent the same situational awareness a senior payments analyst would have after six years on the job.

The practical effect is transformative. Without MCP, an AI assistant looking at a failed payout sees only what you paste into the chat window. With MCP, the agent sees the transaction record, the exception log, the routing rule that applied, the merchant's historical dispute rate, and the runbook for that exception type — simultaneously, in real time. It stops being a chatbot and starts being a fintech agent.

MCP transforms AI from a simple chatbot into a fintech agent that can intelligently reconcile accounts or flag anomalies — because it actually sees the data.

Beyond testing the waters: the advanced tier

The book offers solid starter projects for agentic AI. My focus is further along the maturity curve — the use cases where agentic AI stops being a productivity tool and starts being a core part of the payments processing stack.

1
AI-driven fraud orchestration

Static fraud rules are a known weakness. They are either too aggressive (high false positive rates, merchant friction) or too permissive (fraud slips through). Agentic AI changes the model entirely — instead of rules, you have agents that reason through transaction anomalies based on historical context, merchant behaviour patterns, device fingerprints, and real-time network signals. The agent does not just flag; it explains, and its reasoning is auditable.

2
Autonomous payout workflows

Exception handling in payout workflows is one of the most labour-intensive processes in payments operations. An agent with MCP access can read the exception, identify the root cause, cross-reference the relevant runbook, attempt automated remediation, and escalate only the cases that genuinely require a human — with full context already assembled. The operations team stops being a processing unit and starts being a decision layer.

3
Intelligent clearing file analysis

Clearing files are dense, structured, and unforgiving. Errors are expensive. Using Claude Artifacts and code interpreter tools, you can build agentic workflows that parse clearing files, flag discrepancies against expected settlement amounts, and surface reconciliation gaps before the clearing window closes — not after.

The precision mandate

There is a tension at the heart of applying Vibe Coding to payments. The book celebrates speed and iteration. Payments demands precision and auditability. These are not opposites — but holding both requires deliberate architectural thinking.

My recent IHK Strategic Deep-Dive into Software Architecture was a deliberate step in this direction. As we accelerate with GenAI and agentic workflows, the engineering foundations have to be able to carry the weight. Intent must always be governed by integrity. An AI agent that can initiate a payout must do so within a framework that is auditable, permissioned, and recoverable.

The precision checklist for agentic payments
  • Every agent action is logged with full reasoning trace — not just the outcome
  • Spending and execution limits are defined in the mandate layer, not in the agent
  • MCP connections are permissioned and scoped — agents see only what they need
  • Human escalation paths are designed in, not bolted on
  • Rollback and recovery are first-class requirements, not afterthoughts
  • Regulatory alignment is validated before any agent enters a live processing stack

Vibe Coding accelerates the journey. Architecture ensures we arrive at the right destination. The two are not in conflict — but it takes product leaders who understand both to hold that balance.

In payments, "intent" must always be governed by "integrity." Speed without architecture is just technical debt at scale.
